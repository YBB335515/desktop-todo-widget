# 老师 Skill

## 触发条件
用户说"帮我解释一下"或者使用 `/老师` 命令。

## 行为流程

### Step 1: 确认目标文件
如果用户没有指定文件路径，询问用户：
"好的，你想让我分析哪个文件？请提供文件路径或文件名。"

### Step 2: 读取并分析文件
读取目标文件，分析以下内容：
- **代码整体结构**：主要类/函数/模块及其作用
- **关键依赖**：import 了哪些库，各自用途
- **API 调用**：使用了哪些外部库的 API，参数含义
- **术语**：代码中出现的专业术语和技术名词

### Step 3: 生成 Word 学习笔记
使用 `python-docx` 库创建文档，或者直接运行 `.claude/skills/老师/generate_notes.py <file_path>`。

文档必须包含以下四个章节：

#### 第一章：代码清单（完整代码）
带行号的完整代码，使用等宽字体（Consolas），代码着色使用深灰色。

#### 第二章：逐段白话解释
按功能模块将代码分块，每个块包含：
- **代码片段**：关键代码的引用
- **白话解释**：用通俗易懂的中文解释这段代码在做什么，为什么这样写
- **设计思路**：这种写法解决了什么问题

#### 第三章：API 使用教程
列出代码中使用的所有外部库/API，每个包含：
- **库名和版本**：如 PyAudio、Vosk、tkinter
- **核心函数/类**：代码中用到的具体 API
- **参数说明**：每个参数的含义
- **使用示例**：从代码中提取的实际用法

#### 第四章：术语表
列出代码中涉及的所有技术术语，每个包含：
- **术语**：原文（如 "PortAudio", "KaldiRecognizer"）
- **中文解释**：通俗易懂的中文说明
- **在代码中的作用**：为什么需要它

### Step 4: 保存并告知用户
文档保存为 `{原文件名}_学习笔记.docx`，与源文件在同一目录。
最后告诉用户："笔记生成好了 ✅ 文件路径: {path}"

## 文档格式规范
- 标题：Microsoft 宋体 字体，加粗
- 正文：Microsoft 宋体  字体，11pt
- 代码：Consolas 字体，9pt，深灰色 (#333333)
- 页边距：上下 2cm，左右 2.5cm
